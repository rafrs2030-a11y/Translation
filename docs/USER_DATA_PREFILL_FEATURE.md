# ميزة الملء التلقائي لبيانات المستخدم
## User Data Pre-fill Feature

## نظرة عامة
تم تطوير ميزة جديدة لملء بيانات المستخدم تلقائياً في نموذج تقديم البحث الجديد من الملف الشخصي للمستخدم المسجل.

## التحديثات المنفذة

### 1. تحديث قاعدة البيانات

#### إضافة حقول جديدة إلى جدول `users`
تم إضافة الحقول التالية:
- `gender` (VARCHAR(10)) - الجنس (ذكر/أنثى)
- `country` (VARCHAR(100)) - الدولة

**الملفات المعدلة:**
- `database/schema.sql` - تحديث Schema الرئيسي
- `database/migrations/003_add_user_profile_fields.sql` - Migration جديد

#### تطبيق Migration
لتطبيق التحديثات على قاعدة البيانات، قم بتشغيل:
```sql
-- في Supabase SQL Editor
ALTER TABLE users 
ADD COLUMN IF NOT EXISTS gender VARCHAR(10) CHECK (gender IN ('ذكر', 'أنثى'));

ALTER TABLE users 
ADD COLUMN IF NOT EXISTS country VARCHAR(100);

CREATE INDEX IF NOT EXISTS idx_users_country ON users(country);
```

### 2. تحديث صفحة التسجيل

#### HTML (`public/pages/register.html`)
- إضافة حقل اختيار الجنس (Gender)
- إضافة حقل اختيار الدولة (Country) مع قائمة كاملة بالدول العربية

#### JavaScript (`public/js/auth/register.js`)
- إضافة `gender` و `country` إلى بيانات التسجيل
- إضافة التحقق من صحة هذه الحقول في دالة `validateForm`

### 3. تحديث authStore (`public/js/stores/authStore.js`)
- تحديث دالة `register` لحفظ حقول `gender` و `country` في قاعدة البيانات
- إضافة هذه الحقول إلى Supabase Auth metadata

### 4. تحديث نموذج تقديم البحث (`public/js/researcher/submit.js`)

#### الميزة الرئيسية: الملء التلقائي
تم إضافة دالة `prefillUserData()` التي تقوم بـ:

1. **الحصول على بيانات المستخدم الحالي**
   ```javascript
   const currentUser = await authStore.getCurrentUser();
   ```

2. **ملء الحقول التالية تلقائياً:**
   - الاسم الكامل ← من `username`
   - البريد الإلكتروني ← من `email`
   - رقم الهوية ← من `national_id`
   - الجنس ← من `gender` (إذا كان متوفراً)
   - الدولة ← من `country` (إذا كان متوفراً)

3. **الشروط:**
   - يتم الملء فقط إذا كان الحقل فارغاً
   - لا يتم الكتابة فوق البيانات الموجودة
   - إذا كانت هناك مسودة محفوظة، تأخذ الأولوية

#### سير العمل
```javascript
document.addEventListener('DOMContentLoaded', async () => {
    const user = await requireResearcher();
    if (!user) return;
    
    initElements();
    initEventListeners();
    
    // 1. الملء التلقائي من بيانات المستخدم
    await prefillUserData(user);
    
    // 2. تحميل المسودة (تتجاوز البيانات المملوءة تلقائياً)
    loadDraftIfExists();
});
```

## الفوائد

### 1. تحسين تجربة المستخدم
- توفير الوقت والجهد على المستخدم
- تقليل الأخطاء في إدخال البيانات
- سلاسة أكبر في عملية تقديم البحث

### 2. اتساق البيانات
- نفس البيانات المستخدمة في الملف الشخصي
- تقليل التباين في المعلومات

### 3. سهولة الاستخدام
- المستخدم يمكنه تجاوز الخطوة الأولى بسرعة
- التركيز على تفاصيل البحث الفعلية

## كيفية الاستخدام

### للمستخدمين الجدد
1. سجل حساب جديد في `/pages/register.html`
2. أدخل جميع المعلومات المطلوبة بما في ذلك الجنس والدولة
3. عند تقديم بحث جديد، ستظهر معلوماتك تلقائياً

### للمستخدمين الحاليين
1. قم بتطبيق Migration على قاعدة البيانات
2. حدّث ملفك الشخصي لإضافة الجنس والدولة (قريباً)
3. عند تقديم بحث جديد، ستملأ المعلومات المتاحة تلقائياً

## التطويرات المستقبلية

### 1. صفحة الملف الشخصي
- إضافة حقول Gender و Country للتعديل
- السماح للمستخدمين الحاليين بتحديث معلوماتهم

### 2. البيانات الإضافية
يمكن إضافة المزيد من الحقول للملء التلقائي:
- التخصص العام
- التخصص الدقيق
- المؤسسة التعليمية
- الدرجة العلمية

### 3. الذكاء في الملء
- حفظ آخر قيم مستخدمة لبعض الحقول
- اقتراح قيم بناءً على الأبحاث السابقة

## الاختبار

### اختبار الملء التلقائي
1. سجل مستخدم جديد مع جميع البيانات
2. انتقل إلى صفحة تقديم بحث جديد
3. تحقق من ملء الحقول التالية:
   - الاسم الكامل ✓
   - البريد الإلكتروني ✓
   - رقم الهوية ✓
   - الجنس ✓
   - الدولة ✓

### اختبار المسودة
1. أملأ بعض الحقول بقيم مختلفة
2. احفظ كمسودة
3. أعد تحميل الصفحة
4. تأكد أن المسودة تتجاوز البيانات المملوءة تلقائياً

### اختبار المستخدمين القدامى
1. مستخدم بدون gender/country
2. يجب أن يملأ username, email, national_id فقط
3. gender و country يظلان فارغين

## الأمان والخصوصية

- جميع البيانات تُحمّل من جلسة المستخدم المصادق عليها
- لا يمكن للمستخدم رؤية بيانات مستخدمين آخرين
- البيانات محمية بـ RLS policies في Supabase
- الملء التلقائي يحدث فقط بعد التحقق من الهوية

## الأخطاء والتعامل معها

```javascript
try {
    await prefillUserData(user);
} catch (error) {
    console.error('Error prefilling user data:', error);
    // لا نعرض خطأ للمستخدم
    // النموذج يظل فارغاً ويمكن ملؤه يدوياً
}
```

## الملاحظات التقنية

### الأداء
- الملء التلقائي يحدث بشكل غير متزامن (async)
- لا يؤثر على سرعة تحميل الصفحة
- استخدام `getCurrentUser()` من cache إن أمكن

- لا يكسر الوظائف الحالية
- backward compatible
