# Epic 1: User Authentication System

## Overview
نظام شامل لإدارة المستخدمين يشمل التسجيل، تسجيل الدخول، التحقق من البريد الإلكتروني، واستعادة كلمة المرور.

## Business Value
تمكين الباحثين والمسؤولين من الوصول الآمن إلى المنصة مع ضمان صحة الهويات.

## User Stories

### US-1.1: تسجيل مستخدم جديد
**كباحث**، أريد أن أتمكن من إنشاء حساب جديد حتى أتمكن من تقديم أبحاثي.

**معايير القبول:**
- [ ] نموذج تسجيل يحتوي على: اسم المستخدم، البريد الإلكتروني، رقم الهوية الوطنية/الإقامة، رقم الهاتف، كلمة المرور
- [ ] التحقق من صحة البيانات المدخلة
- [ ] التحقق من عدم تكرار البريد الإلكتروني
- [ ] تشفير كلمة المرور
- [ ] إرسال بريد تحقق إلى المستخدم
- [ ] حفظ البيانات في جدول users في Supabase

**تقديرات:**
- Story Points: 8
- الأولوية: High
- المرحلة: Phase 1

---

### US-1.2: تسجيل الدخول
**كمستخدم مسجل**، أريد أن أتمكن من تسجيل الدخول إلى حسابي حتى أصل إلى لوحة التحكم الخاصة بي.

**معايير القبول:**
- [ ] نموذج تسجيل دخول يحتوي على البريد الإلكتروني وكلمة المرور
- [ ] التحقق من صحة بيانات الدخول
- [ ] إنشاء جلسة آمنة للمستخدم
- [ ] التوجيه إلى لوحة التحكم المناسبة (باحث/مسؤول)
- [ ] عرض رسالة خطأ واضحة عند فشل الدخول

**تقديرات:**
- Story Points: 5
- الأولوية: High
- المرحلة: Phase 1

---

### US-1.3: التحقق من البريد الإلكتروني
**كمستخدم جديد**، أريد أن أتلقى رابط تحقق عبر البريد الإلكتروني حتى أؤكد صحة بريدي.

**معايير القبول:**
- [ ] إرسال بريد إلكتروني تلقائي عند التسجيل
- [ ] رابط تحقق فريد صالح لمدة 24 ساعة
- [ ] صفحة تأكيد التحقق
- [ ] تفعيل الحساب بعد التحقق
- [ ] إمكانية إعادة إرسال البريد

**تقديرات:**
- Story Points: 5
- الأولوية: High
- المرحلة: Phase 1

---

### US-1.4: استعادة كلمة المرور
**كمستخدم**، أريد أن أتمكن من استعادة كلمة المرور إذا نسيتها.

**معايير القبول:**
- [ ] نموذج "نسيت كلمة المرور" يطلب البريد الإلكتروني
- [ ] إرسال رابط إعادة تعيين كلمة المرور
- [ ] صفحة إعادة تعيين كلمة المرور
- [ ] التحقق من قوة كلمة المرور الجديدة
- [ ] تحديث كلمة المرور في قاعدة البيانات

**تقديرات:**
- Story Points: 5
- الأولوية: Medium
- المرحلة: Phase 1

## Technical Requirements

### Database Schema
```sql
-- Users table in Supabase
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  national_id VARCHAR(50) NOT NULL,
  phone VARCHAR(20) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  email_verified BOOLEAN DEFAULT FALSE,
  role VARCHAR(20) DEFAULT 'researcher', -- 'researcher' or 'admin'
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### API Endpoints
- `POST /api/auth/register` - تسجيل مستخدم جديد
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/verify-email` - التحقق من البريد الإلكتروني
- `POST /api/auth/forgot-password` - طلب إعادة تعيين كلمة المرور
- `POST /api/auth/reset-password` - إعادة تعيين كلمة المرور
- `POST /api/auth/logout` - تسجيل الخروج

### Dependencies
- Supabase Auth
- bcrypt للتشفير
- nodemailer للبريد الإلكتروني
- JWT للجلسات
